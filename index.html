Ray Tracing 2: Triangles & Recursion

This project takes my Project 3A raytracer and extends it with full triangle mesh support. Not only that but this project included smooth per-vertex normal interpolation, recursive reflection and refraction, and extra geometric primitives like boxes and planes. The raytracer handles multiple light types (point, directional, and spotlight) with shadow computations and now includes major performance optimizations through BVH acceleration.

Technical Challenges

I ran into a few real struggles while working on this project. Getting BVH acceleration working was a huge challenge. Figuring out how to split triangles and debug the traversal logic took way longer than I expected. I also spent a lot of time wrestling with texture and normal mapping on triangle meshes, especially when mixing smooth shading and different materials. In the end, these struggles taught me a lot about how important it is to both make things fast and get the visuals right.

Bonus Features Implemented (20 points)

+5 points - Spot Lights

Added cone-shaped lights with smooth intensity falloff between inner and outer angles.

+5 points - Boxes and Planes

Added two new primitive types:

  • Boxes: Axis-aligned boxes using min/max corners
  • Planes: Infinite surfaces defined by point and normal

+10 points - BVH Acceleration Structure

Added spatial acceleration structure to speed up ray-triangle intersection tests significantly.

  • Uses binary tree with bounding boxes to organize triangles
  • Reduces intersection tests from O(n) to O(log n)
  • Achieves 10×(+) speedup on complex scenes

Build & Usage Instructions

Compile:

g++ -std=c++17 -O3 -o raytracer main.cpp scene.cpp

Run:

./raytracer scene_file.txt

What you need: A C++17 compiler and the STB image library (already included as stb_image_write.h in the source).

Output: The program writes a BMP file specified by the output_image: line in your scene file. You'll see progress updates in the terminal while it renders, plus timing info when it finishes.
